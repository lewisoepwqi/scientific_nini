## 五、10条最佳实践速查

### 架构（3条）

10. **渐进增强** - MVP→向量→图谱，按需演进

## 六、常见反模式 ⚠️

| 反模式 | 问题 | 正确做法 |
|--------|------|----------|
| 全存不遗忘 | 噪声累积 | 重要性评估+自动遗忘 |
| 纯向量存储 | 缺乏关系 | 向量+图谱混合 |
| 单层架构 | 效率低下 | 三层分层 |
| 无元数据 | 无法筛选 | 必须有时间/情境标签 |
| 硬编码检索 | 不灵活 | 自适应检索策略 |

## 七、快速启动模板

```python
# 三层记忆架构伪代码

class MemorySystem:
    def __init__(self):
        self.working = {}      # 当前上下文
        self.short_term = VectorStore()   # 向量存储
        self.long_term = GraphStore()     # 图数据库

    def store(self, memory, context):
        # 评估重要性
        importance = self.score_importance(memory)

        # 分层存储决策
        if importance > 0.8:
            self.long_term.save(memory, context)
        elif importance > 0.3:
            self.short_term.save(memory, context)
        else:
            self.working[memory.id] = memory

    def retrieve(self, query, context):
        # 层级检索
        results = []
        results.extend(self.working.get(query))           # L1
        results.extend(self.short_term.search(query))     # L2
        results.extend(self.long_term.traverse(query))    # L3

        # 情境重排
        return self.rerank_by_context(results, context)
```

## 八、关键技术参数参考

| 参数 | 建议值 | 说明 |
|------|--------|------|
| 工作记忆容量 | 4-8 轮对话 | 当前上下文窗口 |
| 短期记忆衰减 | 24-72 小时 | 自动降级时间 |
| 向量维度 | 768-1536 | 根据 Embedding 模型 |
| 检索 Top-K | 5-10 | 初步召回数量 |
| 重排后返回 | 3-5 | 最终使用数量 |
| 遗忘阈值 | 0.1-0.3 | 重要性低于则遗忘 |
