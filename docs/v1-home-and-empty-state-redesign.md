# V1 首页与空会话改版方案

本文档用于将 [user-experience-diagnosis-and-optimization.md](/home/lewis/coding/scientific_nini/docs/user-experience-diagnosis-and-optimization.md) 中的 UX 诊断，进一步落成可执行的首页与空会话改版方案。目标不是讨论抽象原则，而是明确 V1 首页应该展示什么、隐藏什么、如何引导用户开始三类首发场景，以及如何把“可信输出、渐进式复杂度、人工复核”前置成可感知的体验。

## 1. 背景

根据 [nini-vision-charter.md](/home/lewis/coding/scientific_nini/docs/nini-vision-charter.md)，Nini 的 V1 不是“全阶段科研自动化系统”，而是聚焦以下三类首发场景的科研协作产品：

1. 数据分析到结果写作衔接
2. 文献综述草稿与证据整理
3. 实验设计中的样本量估计与方案草稿

当前实现中：

- 无会话时，主面板展示 `RecipeCenter + 新建自由会话`
- 已有会话但无消息时，主面板展示 `WelcomeHint + RecipeCenter`
- 全局导航同时暴露会话、知识库、技能、研究画像、分析能力、文章初稿、成本统计、工具清单、设置

相关实现位于：

- [ChatPanel.tsx](/home/lewis/coding/scientific_nini/web/src/components/ChatPanel.tsx)
- [RecipeCenter.tsx](/home/lewis/coding/scientific_nini/web/src/components/RecipeCenter.tsx)
- [GlobalNav.tsx](/home/lewis/coding/scientific_nini/web/src/components/GlobalNav.tsx)
- [AuthGate.tsx](/home/lewis/coding/scientific_nini/web/src/components/AuthGate.tsx)
- [ModelConfigPanel.tsx](/home/lewis/coding/scientific_nini/web/src/components/ModelConfigPanel.tsx)
- [OutputLevelExplainer.tsx](/home/lewis/coding/scientific_nini/web/src/components/OutputLevelExplainer.tsx)

当前问题不是“没有入口”，而是入口还没有按 V1 的产品定位组织。

## 2. 改版目标

本次改版只解决首页和空会话的核心问题，不试图一次性重构整站。

### 2.1 核心目标

| 目标 | 说明 |
| --- | --- |
| 表达 V1 定位 | 用户进入应用后，能立即理解当前版本重点能做什么 |
| 收敛首发场景 | 三类首发场景成为默认起手式，而不是隐藏在众多入口中 |
| 降低认知门槛 | 新用户不需要先理解技能、工具、能力、工作区等系统概念 |
| 支持渐进式复杂度 | 默认只暴露任务启动所需内容，高级能力延后出现 |
| 建立可信心智 | 在首页和空会话阶段就让用户感知“结果有等级、风险有边界、必要时需复核” |

### 2.2 非目标

| 非目标 | 说明 |
| --- | --- |
| 不重写全站导航 | 本方案优先处理首页与空会话，不在本轮重做全部 IA |
| 不处理完整任务执行层 | 统一任务状态中心另立方案 |
| 不处理完整交付中心 | 导出与交付链路另立方案 |
| 不解决所有阶段入口 | 首页只聚焦 V1 三类首发场景与四个重点阶段表达 |

## 3. 设计原则

### 3.1 场景优先，不是功能优先

首页不再首先回答“有哪些功能”，而首先回答“你现在要推进哪类科研任务”。

### 3.2 先完成任务，再理解系统

新用户先完成第一次任务，再逐步接触技能、能力、工作区、成本等高级层。

### 3.3 明确边界，不制造过度承诺

首页和空会话必须明确这是一个“科研协作产品”，不是“自动替你完成研究”的系统。

### 3.4 可信信息前置

输出等级、风险边界、来源与人工复核，不应只在结果页出现，而应在任务启动时就形成预期。

## 4. 当前状态诊断

### 4.1 当前空会话行为

| 状态 | 当前表现 | 问题 |
| --- | --- | --- |
| 无会话 | `RecipeCenter + 或先开一个自由会话` | 有模板，但缺少产品定位表达与场景解释 |
| 有会话但无消息 | `欢迎提示 + RecipeCenter` | 更像功能引导，而不是任务入口页 |
| 全局导航常驻 | 直接暴露大量面板 | 新用户容易被导航分散注意力 |

### 4.2 当前欢迎与引导

当前欢迎文案主要是：

- 欢迎使用 Nini
- 从模板快速开始科研分析，或直接在下方输入需求
- 输入 `/` 发现更多分析技能

这套文案的问题是：

1. 仍然偏“科研分析”定位，没有体现文献调研与实验设计。
2. 太早暴露 slash 技能，不符合渐进式复杂度。
3. 没有体现结果等级、复核边界和证据意识。

### 4.3 当前输出等级提示的问题

虽然已有 [OutputLevelExplainer.tsx](/home/lewis/coding/scientific_nini/web/src/components/OutputLevelExplainer.tsx)，但目前仍存在两个问题：

1. 它更像“首次说明卡”，而不是系统级交互契约。
2. `O4 可导出级` 的文案容易被误解为“最终成果可直接用”，与愿景文档中“仍需人工终审”的边界不完全一致。

结论：

- 首页与空会话不只是要“更好看”，而是要承担产品定位表达、信任建立和任务收敛三重职责。

## 5. 新首页信息架构

### 5.1 一级结构

建议首页与空会话统一采用 5 个区块，自上而下排列：

1. 顶部定位区
2. 三类首发场景区
3. 四个重点阶段说明区
4. 可信协作说明区
5. 历史继续与高级入口区

### 5.2 结构说明

| 区块 | 作用 | 是否默认展开 |
| --- | --- | --- |
| 顶部定位区 | 告诉用户 Nini 当前版本重点帮助什么任务 | 是 |
| 三类首发场景区 | 作为默认主入口，直接开始任务 | 是 |
| 四个重点阶段说明区 | 解释当前版本覆盖范围 | 是 |
| 可信协作说明区 | 解释输出等级、风险边界、人工复核 | 是 |
| 历史继续与高级入口区 | 继续上次、进入自由会话、展开专家能力 | 是，但权重较低 |

## 6. 页面方案

### 6.1 顶部定位区

#### 目标

用一句话把 V1 说清楚。

#### 建议内容

- 标题：`从文献到写作，推进你的科研关键任务`
- 副标题：`当前版本重点支持文献调研、实验设计、数据分析和论文写作中的高频任务。`

#### 辅助说明

用 3 个短标签建立预期：

- `本地优先`
- `结果分级`
- `高风险内容需复核`

#### 设计要求

- 不要使用空泛口号，如“你的全能科研助手”
- 不要暗示“一句话完成整个研究”

### 6.2 三类首发场景区

这是首页最重要的区域，应替代当前单纯的 `RecipeCenter` 卡片感知。

#### 卡片 A：数据分析到结果写作

| 项目 | 内容 |
| --- | --- |
| 标题 | 数据分析到结果写作 |
| 说明 | 上传数据，完成统计分析、图表生成和结果段落草稿 |
| 输入提示 | 数据文件 / 研究问题 / 待比较变量 |
| 输出预期 | 图表、统计结论、结果段落 |
| 风险提示 | 统计解释需人工复核 |
| CTA | 开始分析 |

#### 卡片 B：文献综述草稿与证据整理

| 项目 | 内容 |
| --- | --- |
| 标题 | 文献综述草稿与证据整理 |
| 说明 | 整理检索结果、归纳主题并生成带来源的综述草稿 |
| 输入提示 | 研究主题 / 检索词 / 已有文献 |
| 输出预期 | 文献清单、主题归纳、综述草稿 |
| 风险提示 | 无法验证的内容必须标记不确定性 |
| CTA | 开始整理文献 |

#### 卡片 C：样本量估计与方案草稿

| 项目 | 内容 |
| --- | --- |
| 标题 | 样本量估计与方案草稿 |
| 说明 | 收集关键参数，给出估计结果和研究方案模板草稿 |
| 输入提示 | 研究问题 / 分组设计 / 预期效应量 |
| 输出预期 | 样本量估计、参数说明、方案草稿 |
| 风险提示 | 方案定稿需专业复核 |
| CTA | 开始实验设计 |

#### 交互要求

每张卡片点击后，不直接进入自由聊天，而是进入对应的受控启动流程：

- 数据分析场景：进入上传 + 问题描述
- 文献场景：进入主题与材料输入
- 设计场景：进入参数收集向导

### 6.3 四个重点阶段说明区

这一层不是主入口，而是帮助用户理解 V1 范围。

#### 展示方式

用四段式轻量流程条展示：

`文献调研 → 实验设计 → 数据分析 → 论文写作`

#### 说明文案

建议配一句话：

`Nini 当前优先帮助你推进这四个阶段的关键任务，而不是替你自动完成整个研究。`

### 6.4 可信协作说明区

这是本轮新增的关键区块，当前产品缺少这一层。

#### 必须回答的三个问题

1. 输出有分级吗？
2. 哪些内容有风险，需要复核？
3. 涉及文献和事实时，来源怎么体现？

#### 建议结构

| 模块 | 建议内容 |
| --- | --- |
| 输出等级 | 建议级 / 草稿级 / 可审阅级 / 可导出级 |
| 风险边界 | 高风险内容不会默认被当成最终结论 |
| 来源与证据 | 涉及文献与事实时展示来源与不确定性 |

#### 文案建议

`Nini 会标注结果等级。涉及统计解释、样本量估计、方案定稿等高风险内容时，系统会明确提示人工复核。`

### 6.5 历史继续与高级入口区

这是为了照顾老用户与高阶用户，但视觉权重必须低于场景区。

#### 建议内容

| 入口 | 作用 |
| --- | --- |
| 继续上次任务 | 打开最近会话或最近未完成任务 |
| 进入自由会话 | 面向熟悉产品的用户 |
| 打开专家入口 | 展开技能、工具、工作区、模型等高级能力 |

#### 规则

- `继续上次任务` 应高于 `进入自由会话`
- `专家入口` 默认折叠，不做主按钮

## 7. 空会话状态拆分

建议将当前“无会话”和“有会话但无消息”统一重构为两个明确状态。

### 7.1 状态 A：首次进入 / 无历史任务

#### 展示重点

- 顶部定位区
- 三类首发场景区
- 四阶段说明
- 可信协作说明
- 自由会话次按钮

#### 不应优先展示

- slash 技能引导
- 工具清单
- 工作区说明
- 成本统计

### 7.2 状态 B：已有会话但当前为空

#### 展示重点

- 当前会话所属场景
- 推荐继续动作
- 最近一次相关结果摘要或无结果说明
- 重新启动该场景 / 切换到其他场景

#### 示例

如果该会话是“文献综述草稿”类型，则显示：

- `继续整理来源`
- `生成综述结构`
- `切换到实验设计`

而不是重复展示通用欢迎语。

## 8. 导航与入口权重调整建议

### 8.1 首页阶段的导航处理

| 当前入口 | 建议 |
| --- | --- |
| 会话 | 保留 |
| 知识库 | 保留，但权重降低 |
| 技能 | 下沉到专家入口 |
| 研究画像 | 下沉 |
| 分析能力 | 并入场景表达，不单独做首屏主入口 |
| 文章初稿 | 不单独做一级入口，归入场景链路 |
| 成本统计 | 下沉 |
| 工具清单 | 下沉 |
| 设置 | 保留 |

### 8.2 首页不建议直接强调的内容

以下内容不应在首页主视觉区直接强调：

- `/` slash 技能
- 工具枚举
- 高级模型切换
- 工作区文件管理
- 成本与推理细节

这些内容应该在用户进入任务后再逐步暴露。

## 9. 与当前组件的映射关系

| 当前组件 | 建议处理 |
| --- | --- |
| `RecipeCenter` | 保留，但升级成首页主场景卡容器，而不是单独模板模块 |
| `WelcomeHint` | 取消现有通用欢迎文案，改成基于场景的状态头部 |
| `ChatInputArea` | 在首页和空会话阶段后置，不再成为默认第一动作 |
| `OutputLevelExplainer` | 升级为首页可信协作区的一部分，并同步修正文案边界 |
| `GlobalNav` | 保留结构，但在首页阶段弱化高阶入口权重 |

## 10. 文案方向建议

### 10.1 建议采用的语气

- 专业，但不生硬
- 明确，但不过度承诺
- 强调协作与复核，而不是“全自动”

### 10.2 不建议继续使用的表达

- “从模板快速开始科研分析”
- “试试输入 / 发现更多分析技能”
- “最终成果，可直接用于报告或论文”

问题分别是：

1. 仍把定位缩窄为分析。
2. 过早暴露高阶能力。
3. 容易弱化人工终审边界。

## 11. 验收标准

首页与空会话改版后，至少应满足以下验收项：

| 验收项 | 标准 |
| --- | --- |
| 首屏定位清晰 | 新用户进入后 5 秒内可说出当前版本重点支持的 3 类任务 |
| 主入口收敛 | 三类首发场景均为首屏主入口 |
| 渐进式复杂度 | 新用户不必先理解技能、工具、工作区即可开始任务 |
| 可信边界可见 | 首页可见输出等级、风险边界和复核说明 |
| 高阶入口后置 | 技能、工具、成本、工作区不抢占首页主视觉 |

## 12. 建议的研发拆解顺序

### P0

1. 重构 `ChatPanel` 的无会话与空消息状态。
2. 升级 `RecipeCenter` 为三类首发场景主入口。
3. 新增首页可信协作说明区。
4. 修正输出等级文案和边界表达。

### P1

1. 增加“继续上次任务”模块。
2. 增加场景化受控启动流程。
3. 调整 `GlobalNav` 首页阶段的入口权重。

### P2

1. 增加专家入口与模式切换。
2. 细化不同场景的空会话状态头部。
