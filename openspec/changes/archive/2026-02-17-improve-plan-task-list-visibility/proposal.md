## Why

当前分析计划任务列表不在用户主要注意路径（对话区上方），用户在执行分析时难以及时感知“当前进行到哪一步、还剩哪些步骤、下一步该做什么”，导致理解成本和操作犹豫上升。该问题直接影响分析流程的可控感与完成效率，需要优先优化信息层级与状态可见性。

## What Changes

- 将“分析计划任务列表”调整为位于对话区上方的高可见区域，并在分析过程中持续展示当前进度。
- 引入统一的任务状态表达（未开始、进行中、已完成、阻塞/失败），并提供 `step x/y` 级别的进度标识。
- 在任务列表中增加“当前步骤说明 + 下一步提示”文案，减少用户对系统意图的猜测。
- 为移动端补充折叠与精简展示策略，避免顶部区域挤压主对话内容。
- 增加可观测埋点（如步骤切换、用户回看任务列表行为、流程中断点），用于验证困惑是否下降。

## Capabilities

### New Capabilities
- `analysis-plan-progress-header`: 在对话区上方提供持续可见的分析计划任务列表，支持步骤序号、状态标记、当前步骤高亮与移动端自适应展示。
- `analysis-plan-guidance-copy`: 为任务列表提供面向用户的进度说明与下一步引导文案，覆盖进行中、完成、阻塞/失败等关键状态。

### Modified Capabilities
- `conversation`: 调整会话页面信息层级与布局要求，确保分析任务进度在用户阅读与交互主路径中可见。

## Impact

- 前端：会话页面布局与任务列表组件（`web/src/` 下对话页、计划展示组件、状态样式与响应式逻辑）将发生变更。
- 状态管理：需要调整分析任务状态在前端 store 与 WebSocket 事件消费中的映射关系。
- 观测：新增与任务可见性相关的埋点与日志字段，用于评估优化效果。
- 兼容性：不引入后端 API 破坏性变更；如现有事件字段不足，需以向后兼容方式补充。
