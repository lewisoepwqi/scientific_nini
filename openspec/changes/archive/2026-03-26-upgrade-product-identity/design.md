## Context

Nini 的系统提示词分为三层（见 `docs/prompt-architecture.md`）：

1. **System Prompt**（受信）：由 `src/nini/agent/prompts/builder.py` 装配，读取 `data/prompt_components/` 下的 `identity.md`、`strategy.md`、`security.md`、`workflow.md`、`agents.md`、`user.md`、`memory.md` 等组件。
2. **Runtime Context**（不可信）：由 `src/nini/agent/components/context_builder.py` 注入数据集、技能、知识、画像等运行时信息。
3. **Tool Schema**：工具定义层。

本 change 仅修改第 1 层中的 `identity.md` 和 `strategy.md`，以及仓库根目录的 `CLAUDE.md`。不涉及第 2、3 层和任何 Python/TypeScript 代码。

当前 `identity.md` 为单行文本：
> 你是 Nini，一位专业、严谨、可审计的科研数据分析 AI 助手。

当前 `strategy.md` 包含：7 步标准分析流程 + 任务规划规范 + 沙箱说明 + 可视化决策树 + 绘图规范 + 报告生成决策 + 工作流模板说明。

## Goals / Non-Goals

**Goals:**
- identity.md 反映全流程定位、八大阶段覆盖声明、核心优势声明和责任边界
- strategy.md 在保留现有数据分析策略的前提下，新增通用策略和三个新阶段策略
- CLAUDE.md 项目概述段落与新定位对齐
- 现有数据分析场景的 Agent 行为不退化

**Non-Goals:**
- 不修改 security.md、workflow.md 等其他提示词组件
- 不修改 `builder.py` 的装配逻辑或组件顺序
- 不新增提示词组件文件
- 不做 Token 预算调整（strategy.md 增量内容在合理范围内）

## Decisions

### D1: identity.md 的结构

**选择**：从单行文本扩展为结构化的多段落身份声明。

**内容结构**：
1. 身份声明（一句话：贯穿科研全流程的 AI 研究伙伴）
2. 阶段覆盖声明（列出八大阶段，标注核心优势在数据分析）
3. 责任边界声明（协助研究不替代判断，高风险需人工复核）

**理由**：单行无法承载全流程定位的语义丰富度。结构化多段落可以让 LLM 更稳定地理解覆盖范围和行为边界。

**替代方案**：保持单行但加长 → 语义密度过高，LLM 容易忽略后半段。

### D2: strategy.md 的扩展方式

**选择**：在现有内容**之后**追加新阶段策略，不修改现有数据分析策略的任何文字。

**新增结构**：
```
现有内容（完全保留）
---
通用策略（适用所有阶段）
├── 输出等级标注规范
├── 风险提示触发规则
└── 降级行为规范

文献调研策略
├── 检索 → 筛选 → 综合 → 输出
└── 证据溯源要求 + 离线降级提示

实验设计策略
├── 问题定义 → 设计选择 → 参数计算 → 方案生成
└── 伦理提示 + 人工复核提醒

论文写作策略
├── 结构规划 → 分节撰写 → 修订 → 格式化
└── 引用规范 + 草稿级标注
```

**理由**：追加而非重写，最大限度保证数据分析行为不退化。现有 7 步流程经过充分验证，不应触碰。

**替代方案**：重构为模块化多文件（每阶段一个文件）→ 需要修改 `builder.py` 装配逻辑，超出本 change 范围，且会引入不必要的代码变更风险。

### D3: CLAUDE.md 的修改范围

**选择**：仅修改「项目概述」段落的一句话定位描述。

**理由**：CLAUDE.md 是开发者指引文件，其他内容（命令、架构、规范）不受定位升级影响。最小改动原则。

### D4: 新阶段策略的详细程度

**选择**：V1 阶段策略以框架性指导为主，不做到数据分析策略那样的详细程度（如具体的方法选择表、陷阱列表）。

**理由**：新阶段能力尚未实现（L1/L2 级别），过于详细的策略指导没有对应的工具链支撑，反而可能误导 LLM 尝试调用不存在的工具。策略详细程度应与能力成熟度匹配。

## Risks / Trade-offs

- **[风险] identity.md 变长导致 Token 占用增加** → 预计增加 ~200 tokens，在 System Prompt 总预算内可接受。builder.py 已有组件级预算保护机制。
- **[风险] strategy.md 追加内容可能干扰数据分析策略的遵从度** → 通过在新阶段策略前加明确的分隔标记和条件触发说明来缓解（"当用户任务属于文献调研阶段时，参考以下策略"）。
- **[风险] 新阶段策略描述的能力尚未实现，LLM 可能尝试超范围操作** → 在策略中明确标注当前可用工具范围和能力边界，避免 LLM 幻想不存在的工具。
- **[回滚]** 三个文件均为纯文本，git revert 即可完全恢复，零代码侵入。
