# Impeccable Design Context

> This file provides persistent design guidance for all UI/UX work on the Nini project.
> Generated from codebase analysis and user interview.

## Design Context

### Users
- **Who**: 科研人员（研究生、博士后、教授、数据分析师），具备统计学基础但不一定是编程专家
- **Context**: 桌面端长时间工作场景，单次使用可能持续数小时进行数据分析、图表制作、报告撰写
- **Job to be done**: 通过对话上传数据并完成统计分析、可视化、代码执行和报告生成的全流程，核心痛点在数据分析阶段的效率和质量
- **Emotional needs**: 需要对分析结果有信心（信任感），操作流程不中断（沉浸感），AI 的输出要专业准确（专业感）

### Brand Personality
- **Three words**: 专业可信 · 简洁高效 · 现代智能
- **Voice**: 专业但不冷淡，像一个靠谱的科研搭档；用中文沟通，术语保留英文原文
- **Emotional goals**: 用户应该感到"这个工具懂我的研究需求"，而非"这是一个通用聊天机器人"
- **What it is NOT**: 不是花哨的消费级应用，不是冰冷的企业软件，不是玩具式的 AI 演示

### Aesthetic Direction
- **Visual tone**: 精致的产品感（Notion/Linear 级别），克制的装饰，注重排版和间距的呼吸感
- **References**: Notion（信息架构清晰、排版考究）、Linear（精致的暗色模式、流畅的交互细节）
- **Anti-references**: 避免花哨渐变、过度的圆角/阴影、卡通化图标、营销式视觉噪音
- **Theme**: 亮色 + 深色双主题，两者需同等质量设计（研究人员可能深夜使用）
- **Layout philosophy**: 三栏布局（会话列表 + 对话面板 + 工作区），信息密度适中，关键数据一目了然

### Color System
- **Primary palette**: 基于 Tailwind slate 色阶（当前已广泛使用），作为中性色基础
- **Accent semantics** (已在代码中使用的功能色彩，需保持一致):
  - `purple-600`: 分析能力（Sparkles）
  - `emerald-600`: 报告/文章（FileText）
  - `amber-600`: 成本统计（Coins）
  - `sky-600`: 研究画像（User）
  - `blue-600`: 工作区面板（PanelRight）
  - `indigo-600`: 知识库（Library）
- **Status colors**: emerald(连接) / sky(连接中) / amber(警告) / red(断开) / gray(静默)
- **Code blocks**: 深色背景 (`#1e293b`) + 浅色文字，亮暗主题下保持一致
- **Thinking shimmer**: slate 色系渐变动画，传达 AI 思考状态

### Typography
- **Font stack**: 系统字体栈（`-apple-system, BlinkMacSystemFont, Segoe UI, Roboto, Helvetica Neue, Arial, sans-serif`）
- **Scale**: Tailwind defaults, code blocks `0.875em`
- **Markdown rendering**: 自定义样式（见 `index.css`），标题层级 `1.25rem → 1rem`

### Current Tech Stack (Design-Related)
- React 18 + TypeScript + Tailwind CSS 3 + Zustand
- Lucide React for icons
- Plotly.js for data visualization
- React Markdown + remark-gfm for content rendering
- No component library (fully custom)
- `darkMode: 'class'` configured in Tailwind（暗色模式基础设施已就绪）

### Design Principles

1. **Clarity over decoration** — 研究人员需要专注于数据和结论，UI 应该是透明的。避免视觉噪音干扰思考。

2. **Dense but breathable** — 长时间桌面使用需要足够的信息密度（图表、表格、代码、对话），但通过留白和间距让内容有呼吸感。参考 Linear 的信息密度。

3. **Consistent color semantics** — 每个功能域有固定的色彩标识（紫=能力、绿=报告、橙=成本...），用户形成直觉后无需阅读标签即可识别。

4. **Dual-mode parity** — 亮色和深色模式是同一设计的两面，而非暗色是亮色的附属品。深夜使用深色模式是真实场景。

5. **Professional warmth** — 专业但不冰冷。AI 是研究伙伴而非工具，交互文案应有温度，但视觉呈现保持克制和可信。

### Interaction Patterns
- **Primary interaction**: 对话式（WebSocket 流式传输），用户输入 → AI 流式响应
- **Output levels**: o1(建议级/slate) → o2(草稿级/sky) → o3(可审阅级/emerald) → o4(可导出级/violet)
- **Workspace**: 右侧可调宽度面板（340-960px），拖拽分隔条调整
- **Mobile**: 覆盖式侧边栏抽屉，桌面端响应式折叠
- **Tool calls**: 折叠展示，可展开查看详情
- **Charts**: Plotly 交互式图表，懒加载

### Accessibility Notes
- 需要考虑长时间使用的视觉舒适度（眼疲劳）
- 统计图表需要色盲友好的调色板
- 中文内容的排版需保证可读性（行高、字间距）
