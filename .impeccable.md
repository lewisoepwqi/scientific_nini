## Design Context

### Users
- **Who**: 科研人员（研究生、博士后、教授、数据分析师），具备统计学基础但不一定是编程专家
- **Context**: 桌面端长时间工作场景，单次使用可能持续数小时进行数据分析、图表制作、报告撰写
- **Job to be done**: 通过对话上传数据并完成统计分析、可视化、代码执行和报告生成的全流程，核心痛点在数据分析阶段的效率和质量
- **Emotional needs**: 需要对分析结果有信心（信任感），操作流程不中断（沉浸感），AI 的输出要专业准确（专业感）

### Brand Personality
- **Three words**: 专业可信 · 简洁高效 · 现代智能
- **Voice**: 专业但不冷淡，像一个靠谱的科研搭档；用中文沟通，术语保留英文原文
- **Emotional goals**: 用户应该感到"这个工具懂我的研究需求"，而非"这是一个通用聊天机器人"
- **What it is NOT**: 不是花哨的消费级应用，不是冰冷的企业软件，不是玩具式的 AI 演示

### Aesthetic Direction
- **Visual tone**: 精致的产品感（Linear 级别），克制的装饰，注重排版和间距的呼吸感
- **References**: Linear（精致的暗色模式、极致密度、精准间距、流畅交互细节）、Notion（信息架构清晰、排版考究）
- **Anti-references**: 避免花哨渐变、过度的圆角/阴影、卡通化图标、营销式视觉噪音
- **Theme**: 暗色模式优先（深夜科研场景），亮色模式需同等质量
- **Layout philosophy**: 三栏布局（会话列表 240px + 对话面板 flex-1 + 工作区按需展开），信息密度适中，关键数据一目了然

## Color System (v1.2.0)

> 来源：`design-system/MASTER.md` §2 Color Tokens
> ⚠️ 所有颜色必须通过 CSS token 引用，**严禁硬编码 hex 值或 Tailwind 原始色彩类名**

### 双模式实现方案

项目使用 Tailwind `darkMode: 'class'`，通过在 `<html>` 元素上切换 `.dark` class 实现模式切换。所有颜色 token 在 `index.css` 中声明，`:root` 为亮色默认值，`.dark` 覆盖为暗色值。

```css
/* index.css — 完整双模式 token 声明 */

  /* === 亮色模式（Light Mode）=== */

  /* 背景层级 */
  --bg-app:      #F5F5F6;
  --bg-base:     #FFFFFF;
  --bg-elevated: #F0F0F2;
  --bg-overlay:  #E8E8EA;
  --bg-hover:    #EBEBED;

  /* 边框 */
  --border-subtle:  #E8E8EA;
  --border-default: #DCDCE0;
  --border-strong:  #C8C8CE;

  /* 文字 */
  --text-primary:   #1A1A1C;
  --text-secondary: #6A6A6F;
  --text-muted:     #9A9AA0;
  --text-disabled:  #C0C0C6;

  /* 强调色（亮色/暗色模式保持一致，这是 Linear 的特点） */
  --accent:        #5E6AD2;
  --accent-hover:  #4E5BC2;
  --accent-subtle: rgba(94, 106, 210, 0.12);

  /* 功能色（亮色模式加深以保证对比度） */
  --success: #2D7A52;
  --warning: #A06030;
  --error:   #A03838;

  /* 功能域色标（亮色模式加深） */
  --domain-analysis:  #6D28D9;
  --domain-report:    #047857;
  --domain-cost:      #B45309;
  --domain-profile:   #0369A1;
  --domain-workspace: #1D4ED8;
  --domain-knowledge: #4338CA;
}

.dark {
  /* === 暗色模式（Dark Mode）=== */

  /* 背景层级 */

  --bg-base:     #141415;
  --bg-elevated: #1A1A1C;

  /* 功能色 */
```

## Layout Structure（v1.1.0 新增）

### 三栏结构总览

```
┌─────────────┬──────────────────────────────┬───────────────┐
│  Sidebar    │       Main Workspace         │  DetailPanel  │
│  240px      │       flex-1                 │  320px        │
│  固定宽度    │  ┌────────────────────────┐  │  按需显示      │
│  overflow-y │  │  Toolbar    44px       │  │  推入式（中栏  │
│  : auto     │  ├────────────────────────┤  │  相应收缩）    │
│             │  │  ContentArea           │  │               │
│             │  │  flex-1                │  │               │
│             │  │  overflow-y: auto      │  │               │
│             │  ├────────────────────────┤  │               │
│             │  │  InputArea             │  │               │
│             │  │  sticky bottom         │  │               │
│             │  │  （仅对话页）           │  │               │
│             │  └────────────────────────┘  │               │
└─────────────┴──────────────────────────────┴───────────────┘
```

### 各区域行为规范

#### Sidebar（左栏）
- 宽度：`240px` 固定，不可调整
- 高度：`100vh`，`overflow-y: auto`
- 背景：`var(--bg-base)`
- 右边框：`1px solid var(--border-subtle)`
- 折叠：移动端隐藏（`< 768px`），通过 hamburger 触发

#### Main Workspace（中栏）
- 宽度：`flex: 1`，最小宽度 `480px`
- 高度：`100vh`，内部三层各自独立处理溢出
- **Toolbar**（固定层）
  - 高度：`44px`，`flex-shrink: 0`
  - 背景：`var(--bg-base)`
  - 下边框：`1px solid var(--border-subtle)`
  - 左侧：面包屑或页面标题（`text-md font-medium text-primary`）
  - 右侧：操作按钮组（ghost button，`28×28px`）
  - 禁止放入：搜索框、大型 Tab、状态信息
- **ContentArea**（滚动层）
  - `flex: 1`，`overflow-y: auto`
  - `padding: 0`（内部组件自行管理 padding）
  - 列表页：每行 item `height: 36px`，hover `bg-[--bg-hover]`
  - 卡片页：`gap: 12px`，卡片 `padding: 16px`
- **InputArea**（仅对话页，吸底层）
  - `position: sticky; bottom: 0`
  - 背景：`var(--bg-base)`
  - 上边框：`1px solid var(--border-subtle)`
  - padding：`12px 16px`
  - 输入框最大高度 `200px`，超出内部滚动

#### DetailPanel（右栏）
- 宽度：`320px`，按需展示
- 展示方式：**推入式**——DetailPanel 出现时中栏宽度相应收缩，不覆盖主内容
- 高度：`100vh`，`overflow-y: auto`
- 背景：`var(--bg-base)`
- 左边框：`1px solid var(--border-subtle)`
- 顶部 header：`44px`（与 Toolbar 对齐），含标题 + close button
- **无背景遮罩**，不阻断主区域交互
- 内部 section 用 `border-top: 1px solid var(--border-subtle)` + 小标题区隔

**宽度例外（已审批）：**

| 组件 | 宽度 | 例外原因 |
|------|------|---------|
| `CitationPanel` | `384px` | 引用内容需同时展示来源文件名、引用片段和上下文定位，320px 截断严重影响可读性；其他 DetailPanel 一律 320px |

#### 工作区分隔条
- 中栏与右栏之间可拖拽调整
- 拖拽范围：右栏宽度 `240px–480px`
- 拖拽手柄：`4px` 宽，hover 时显示 `var(--accent)` 色

- 分区展示用户研究方向、常用工具、历史分析偏好
- 编辑态：CommandSheet 右侧滑入

## Components (v1.1.0)

### 关键尺寸（强制）

| 组件 | 尺寸 | 字号 |

| Button | 高 28px | 12px |
| Input | 高 28px | 13px |

### Button 规范

- 高度 28px，字号 12px，内边距 `0 12px`，圆角 6px，字重 500
- `transition: background 0.1s ease, color 0.1s ease`（所有 variant 必须）
- Primary：`var(--accent)` 背景 / `#FFF` 文字 / hover `var(--accent-hover)`
- Secondary：`transparent` 背景 / `var(--text-primary)` 文字 / `var(--border-default)` 边框 / hover `var(--bg-hover)`
- Ghost：`transparent` 背景 / `var(--text-secondary)` 文字 / 无边框 / hover `var(--bg-hover)` + `var(--text-primary)`
- Danger：`var(--error)` 背景 / `#FFF` 文字
- Icon Button（Ghost 变体）：`28×28px`，`padding: 0`，图标居中

### Input 规范

- 高度 28px，字号 13px，内边距 `0 8px`，圆角 6px
- 边框 `1px solid var(--border-default)`，背景 `var(--bg-base)`
- focus：`border-color: var(--accent)` + `box-shadow: 0 0 0 1px var(--accent)`
  > ✅ **例外说明**：此处 `box-shadow` 为 focus ring，不属于 Anti-patterns 中「彩色阴影」的限制范围。focus ring 是无障碍必须项，不得删除。

### Toast 规范

- 位置：屏幕右下角，距边缘 16px，最大宽度 360px
- 圆角 6px，内边距 `12px 16px`，字号 13px
- 背景 `var(--bg-elevated)`，边框 `var(--border-default)`
- 变体：左侧 3px 色条（`--success` / `--warning` / `--error`）
- 自动消失：success 3s，warning 5s，error 需手动关闭

### Empty State 规范

- 居中布局，图标 24px（`var(--text-muted)`）
- 标题 14px `var(--text-secondary)`，描述 13px `var(--text-muted)`
- 可选：主操作 Button（primary variant）

## Chat Components（v1.3.0 新增）

> 对话页专属组件，存放于 `web/src/components/chat/`

### TaskTree（折叠式任务树）

> ⚠️ **禁止使用线性 StepProgressBar**。LLM 规划的任务数量不固定（3–20+），可动态追加，线性步骤条无法适应此场景。

**位置：** InspectorPanel 的 Task Progress section 内，或作为 AgentMessageCard 的 header 摘要条

**默认状态（折叠）：**
```
[⟳] 分析两种药物候选物对症状减轻的影响  75%  ▾
```
- 单行，高度 36px
- 左侧：当前步骤状态图标（动态）
- 中间：任务总标题，`text-base font-medium text-primary`
- 右侧：整体完成百分比（`text-sm text-secondary`）+ 展开箭头

**展开状态（任务列表）：**
```
▼ 分析两种药物候选物对症状减轻的影响  75%

  ✓  Define Variables                    Done
  ⟳  Analyze Pathways          ←高亮    In Progress
  ○  Generate Report                     Pending
  ○  Statistical Validation    ←动态追加 Pending
```
- 每个任务项行高 32px，左侧缩进 16px
- 状态图标规范：

| 状态 | 图标（Lucide） | 颜色 |
|------|--------------|------|
| Done | `Check` | `var(--success)` |
| In Progress | `Loader2`（旋转动画） | `var(--accent)` |
| Pending | `Circle` | `var(--text-muted)` |
| Error | `AlertCircle` | `var(--error)` |
| Skipped | `MinusCircle` | `var(--text-disabled)` |

- 当前执行中的任务行：背景 `var(--accent-subtle)`，文字 `var(--accent)`

## Interaction Patterns（v1.1.0 更新）

### 弹窗系统

| 类型 | 宽度 | 位置 | 遮罩 | 适用场景 |
|------|------|------|------|---------|
| **DetailPanel** | 320px（默认） | 右侧固定，推入式 | ❌ 无 | 详情查看、属性面板；不中断主区操作（研究画像入口、工具详情、Agent 信息） |
| **CommandSheet** | 40vw（min 480px） | 右侧滑入 | ✅ 半透明 | 表单操作、新建/编辑（新建 Agent、工具配置、知识库上传） |
| **ConfirmDialog** | ≤ 400px | 居中 | ✅ 半透明 | 删除确认、不可逆操作确认 |

### 弹窗使用决策树

```
用户触发操作
    │
    ├─ 只是「查看」信息？
    │       └─ DetailPanel（推入式，无遮罩）
    │
    ├─ 需要「填写表单」或「配置」？
    │       └─ CommandSheet（右侧滑入，有遮罩）
    │
    └─ 需要「二次确认」不可逆操作？
            └─ ConfirmDialog（居中，有遮罩，≤400px）
```

### 弹窗动画规范

| 类型 | 进入动画 | 时长 |
|------|---------|------|
| DetailPanel | `transform: translateX(100%) → 0` | 150ms ease |
| CommandSheet | `transform: translateX(100%) → 0` | 200ms ease |
| ConfirmDialog | `opacity: 0 → 1` + `scale: 0.96 → 1` | 150ms ease |

### 关键规则
- **禁止**：全屏 Modal、左侧 Drawer、底部 Drawer、原生 Alert
- hover transition：`0.1s ease`
- 所有异步操作必须有 loading 状态（按钮 disabled + spinner）
- 错误反馈用 Toast（非 Alert）；网络断开用顶部 Banner
- ESC 键关闭所有弹窗类组件，焦点回到触发元素
- 空状态必须有专属 Empty State 组件，不显示空白区域
- 工作区分隔条：DetailPanel 宽度 `240px–480px` 可拖拽

### Primary Interaction（对话页专属）
- 传输方式：WebSocket 流式传输，用户输入 → AI 流式响应
- Output levels：`o1`（建议级/slate）→ `o2`（草稿级/sky）→ `o3`（可审阅级/emerald）→ `o4`（可导出级/violet）
- Tool calls：折叠卡片展示，可展开查看入参/出参详情
- Charts：Plotly 交互式图表，懒加载，图表容器保持最小高度防止布局跳动

## Design Principles

1. **极致密度** — 信息以最高效率排列，参考 Linear 的列表密度与面板紧凑度。
2. **冷色中性** — 界面色调冷静中性，强调色仅在关键操作点出现。
3. **精准间距** — 所有间距遵循 4px 基准网格，组件尺寸精确到像素。
4. **克制动效** — `0.1s ease` hover 过渡，无弹跳、缩放、多余动画。
5. **Dual-mode parity** — 暗色模式是首要设计目标，亮色模式需同等质量。亮色模式不是「反色」，而是独立调校的色彩系统（见 Color System v1.2.0）。
6. **Consistent color semantics** — 每个功能域固定色彩标识（见功能域 token），用户形成操作直觉。
7. **Professional warmth** — 专业但不冰冷，交互文案有温度，视觉呈现克制可信。
8. **不中断主流程** — 查看详情类操作使用 DetailPanel（推入式），不打断用户当前任务上下文。

## Anti-patterns

### 绝对禁止
- ❌ 全屏 Modal / 左侧 Drawer / 底部 Drawer / 原生 Alert
- ❌ Emoji 作为图标（统一使用 Lucide React）
- ❌ 弹跳/缩放/旋转动画
- ❌ 渐变背景（用纯色 token）
- ❌ 大于 8px 的圆角用于小组件（badge、tag、button）
- ❌ 彩色 `box-shadow`（**例外：Input focus ring** 使用 `box-shadow: 0 0 0 1px var(--accent)` 是合法的无障碍实践）
- ❌ 硬编码 hex 颜色值或 Tailwind 原始色彩类（如 `text-purple-600`），必须使用 token
- ❌ 功能域颜色使用 Tailwind 类名（如 `text-sky-600`），必须使用 `var(--domain-*)` token
- ❌ 在亮色模式下使用暗色模式的 hex 值（如直接写 `#1A1A1C`），必须通过 token 自动切换
- ❌ Plotly 图表硬编码颜色，必须使用 `getPlotlyLayout(isDark)` 动态配置

### 高度警惕
- ⚠️ 布局偏移型 hover（禁止 `translateY`、`scale` 导致的布局跳动）
- ⚠️ 非 4px 倍数的间距值
- ⚠️ 非 28px 高度的 Button / Input
- ⚠️ 无 loading 状态的异步操作
- ⚠️ DetailPanel 与 CommandSheet 场景混用（查看类必须用 DetailPanel，不用 CommandSheet）
- ⚠️ `prefers-reduced-motion` 未尊重（所有动画必须在此 media query 下禁用）
- ⚠️ 亮色模式下对比度不足（文字对比度需 ≥ 4.5:1，使用加深后的功能域色标）
- ⚠️ 模式切换时出现 FOUC（必须在 `<head>` 内联防闪烁脚本）

## Theme Switching（v1.2.0 新增）

### 模式切换机制

```tsx
// src/stores/themeStore.ts
type Theme = 'light' | 'dark' | 'system'

// 切换逻辑
function applyTheme(theme: Theme) {
  const isDark =
    theme === 'dark' ||
    (theme === 'system' && window.matchMedia('(prefers-color-scheme: dark)').matches)

  document.documentElement.classList.toggle('dark', isDark)
  localStorage.setItem('theme', theme)
}

// 初始化（防止 FOUC，必须在 <head> 内联执行）
const saved = localStorage.getItem('theme') ?? 'system'
applyTheme(saved as Theme)
```

### 防止 FOUC（Flash of Unstyled Content）

在 `index.html` 的 `<head>` 中**内联**以下脚本，必须在任何 CSS 加载之前执行：

```html
<script>
  (function() {
    var theme = localStorage.getItem('theme') || 'system';
    var isDark = theme === 'dark' ||
      (theme === 'system' && window.matchMedia('(prefers-color-scheme: dark)').matches);
    if (isDark) document.documentElement.classList.add('dark');
  })();
</script>
```

### Tailwind 双模式写法规范

```tsx
// ✅ 正确：使用 CSS token（自动响应模式切换）
<div className="bg-[var(--bg-base)] text-[var(--text-primary)] border border-[var(--border-default)]">

// ✅ 正确：Tailwind dark 修饰符（仅用于无法 token 化的场景）
<div className="bg-white dark:bg-[#141415]">

// ❌ 错误：硬编码颜色（模式切换无效）
<div className="bg-[#141415] text-[#E8E8EA]">

// ❌ 错误：Tailwind 原始色彩类（不走 token，双模式失效）
<div className="bg-gray-950 text-gray-100">
```

### 模式切换 UI 规范

- 切换开关位置：Sidebar 底部或 Settings 页
- 支持三态：`light` / `dark` / `system`（跟随系统，默认值）
- 切换动画：无需过渡动画（Linear 也是瞬切）
- 图标：`Sun`（亮色）/ `Moon`（暗色）/ `Monitor`（系统）

| 场景 | 暗色模式 | 亮色模式处理 |

## Accessibility Notes
- 长时间使用的视觉舒适度（防眼疲劳：暗色模式低对比度背景）
- 统计图表需色盲友好调色板（不依赖纯色差区分）
- 中文排版保证可读性（行高 ≥ 1.5，字间距正常）
- Focus 状态必须可见（keyboard a11y，focus ring 不可删除）
- `prefers-reduced-motion` 必须尊重（所有 CSS transition/animation 在此 query 下 duration: 0）
- DetailPanel 打开时焦点移入面板，关闭时焦点返回触发元素
